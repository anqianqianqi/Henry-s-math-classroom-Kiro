# UI/UX Style Guide

## Design Principles
- **Simple & Clean**: Avoid clutter, focus on essential features
- **Intuitive**: Users should understand functionality without instructions
- **Responsive**: Works on desktop, tablet, and mobile
- **Accessible**: Follow WCAG guidelines

## User Roles & Views

### Teacher (Henry)
- Dashboard with class overview
- Create/edit daily challenges
- Assign challenges to classes
- View student submissions
- Upload class materials
- Manage class schedules

### Student
- View assigned daily challenges
- Submit responses
- See community responses (after posting)
- Access class materials
- View class schedule

### Parent
- View child's classes
- See class introduction/schedule
- View child's progress
- Access class materials

## Key UI Patterns

### Daily Challenge
- Card-based layout
- Clear call-to-action for submission
- "Post to unlock community responses" messaging
- Discussion thread view after posting

### Class Materials
- Organized by type (homework, notes, recordings)
- Easy upload for teachers
- Download/view for students and parents

### Notifications
- Non-intrusive
- Clear and actionable
- Dismissible
